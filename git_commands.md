# Справочник по Git-командам

| Команда | Что делает | Когда использовать |
|---------|-----------|-------------------|
| git pull | Получить последние изменения из удаленного репозитория и объединить с локальной веткой | В начале рабочего дня, перед тем как начать работу |
| git checkout -b feature/название | Создать новую ветку и переключиться на неё | Когда начинаете новую задачу |
| git checkout название-ветки | Переключиться на другую ветку | Когда нужно переключиться между ветками в проекте |
| git status | Показать статус текущего репозитория (какие файлы изменены, добавлены и т.д.) | Часто, чтобы видеть текущее состояние репозитория |
| git add файл.txt | Добавить конкретный файл в staging area | Когда хотите добавить конкретные изменения в commit |
| git add . | Добавить все изменённые файлы в staging area | Когда хотите добавить все изменения сразу |
| git commit -m "Описание" | Зафиксировать изменения из staging area в репозитории с сообщением | После добавления файлов в staging area, чтобы зафиксировать изменения |
| git fetch origin master | Получить последние изменения из удалённой ветки master (без объединения) | Перед rebase, чтобы получить свежие данные с сервера |
| git rebase master | Переместить свои коммиты поверх актуальной версии master, создав линейную историю | Перед push, чтобы выстроить коммиты поверх актуальной master |
| git push origin feature/название | Отправить свою ветку в удалённый репозиторий | Когда закончили работу и готовы отправить ветку на сервер |
| git branch | Показать список локальных веток | Когда нужно посмотреть, какие ветки есть локально |
| git branch -a | Показать все ветки (локальные и удалённые) | Когда нужно посмотреть все ветки (включая удалённые) |
| git branch -v | Показать локальные ветки и последний коммит на каждой | Когда нужно посмотреть, какие коммиты на каждой ветке |
| git branch -d название-ветки | Безопасно удалить ветку (только если изменения уже слиты) | Когда ветка больше не нужна и её можно удалить |
| git branch -D название-ветки | Принудительно удалить ветку без проверок | Когда нужно удалить ветку, которая не до конца слита |
| git rebase -i HEAD~N | Открыть интерактивный режим для редактирования последних N коммитов | Когда нужно объединить, переименовать или удалить коммиты |
| git log | Показать полную историю коммитов | Когда нужно посмотреть полную историю проекта |
| git log --oneline | Показать историю коммитов в компактном виде (одна строка на коммит) | Когда нужна компактная история без лишних деталей |
| git log --oneline --graph | Показать историю с графом веток в компактном виде | Когда нужно визуально увидеть, как ветки разветвляются и объединяются |
| git log -N | Показать последние N коммитов в истории | Когда нужно посмотреть ограниченное количество последних коммитов |
| git checkout -- файл.txt | Откатить конкретный файл к состоянию последнего коммита | Когда случайно отредактировали файл и хотите вернуть исходную версию |
| git reset --soft HEAD~1 | Откатить последний коммит, сохранив изменения в staging area | Когда нужно откатить последний коммит, но оставить изменения для правки |
| git reset --mixed HEAD~1 | Откатить последний коммит, сохранив изменения в рабочей директории | Когда нужно откатить коммит, но продолжить работу с изменениями |
| git reset --hard HEAD~1 | Откатить последний коммит и удалить все изменения (осторожно!) | Когда нужно полностью удалить последний коммит и все его изменения |
| git reset --hard хеш-коммита | Откатиться на конкретный коммит, удалив все изменения после него | Когда нужно откатиться на старый коммит (используйте с осторожностью) |
| git reflog | Показать полную историю всех действий в git (история команд, откатов и т.д.) | Когда потеряли коммит или ветку и нужно её восстановить |
| git cherry-pick хеш-коммита | Применить конкретный коммит из другой ветки к текущей | Когда нужно применить коммит из другой ветки к текущей |
| git rebase --continue | Продолжить процесс rebase после разрешения конфликтов | После разрешения конфликтов во время rebase |
| git rebase --abort | Отменить весь процесс rebase и вернуться в исходное состояние | Когда хотите отменить rebase и вернуться назад |
| git diff | Показать различия между файлами в рабочей директории и staging area | Когда нужно посмотреть, что именно изменилось в файлах |
